## Функции

### Предварительное условие

Этот мод **активируется только при обнаружении мода Create (Механическая сила)**. Если Create не установлен, мод будет полностью молчаливым и не будет выполнять никаких действий.

### Режим 1: Фальшивый режим загрузки (по умолчанию)

При входе в мир автоматически появляется диалоговое окно с системным предупреждением о загрузке:

![Демо фальшивого режима](image/gif/Prank.gif)

- Красная предупреждающая рамка + красная строка заголовка «Системное экстренное предупреждение»
- Бирюзовый текст «Обнаружен мод Create (Механическая сила)»
- 8 страшных сообщений, меняющихся каждые 5 секунд
- Цвет полосы прогресса меняется: зелёный → жёлтый → оранжевый → красный
- **8% вероятность регресса прогресса**, имитация сетевого лага
- **Прогресс блокируется на 99%**, никогда не достигает 100%
- Поддельная информация о файле: «Установщик Genshin v5.0.0 - 23.8 ГБ»
- Поддельный обратный отсчёт оставшегося времени
- Можно закрыть кнопкой «Закрыть загрузку» или клавишей ESC

### Режим 2: Реальный режим загрузки (включается через конфигурацию)

После включения в конфигурации всплывающее окно становится настоящим менеджером загрузок:

![Демо реального режима загрузки](image/gif/Download.gif)

- Профессиональный интерфейс загрузки с синей рамкой
- Бирюзовый текст «Обнаружен мод Create (Механическая сила)»
- Асинхронная загрузка в фоновом потоке, **не блокирует основной поток игры**
- Отображение в реальном времени: процент загрузки, загружено/общий размер, скорость (КБ/с), оставшееся время
- Цвет рамки меняется в зависимости от состояния: синий (загрузка) → зелёный (завершено) → красный (ошибка)
- Автооткрытие установщика после загрузки (кроссплатформенно: Windows/macOS/Linux)
- Всплывающее окно ошибки при таймауте сети, проблемах с правами, ошибке загрузки
- Поддержка отмены загрузки

## Установка

1. Установите [Fabric Loader](https://fabricmc.net/use/installer/) (если ещё не установлен)
2. Скачайте и установите [Fabric API](https://modrinth.com/mod/fabric-api) → поместите в `.minecraft/mods/`
3. Скачайте и установите [Fabric Language Kotlin](https://modrinth.com/mod/fabric-language-kotlin) → поместите в `.minecraft/mods/`
4. Скачайте и установите [Create](https://modrinth.com/mod/create) → поместите в `.minecraft/mods/`
5. Поместите `create-download-genshin-1.0.0.jar` в `.minecraft/mods/`
6. Запустите игру и войдите в любой мир, чтобы увидеть всплывающее окно

## Конфигурация

При первом запуске мод автоматически создаёт файл конфигурации. Если файл конфигурации повреждён или поля отсутствуют, значения автоматически восстанавливаются по умолчанию.

**Путь к файлу конфигурации:**
```
.minecraft/config/create-download-genshin/mod_config.json
```

### Конфигурация по умолчанию

```json
{
  "enableRealDownload": false,
  "downloadUrl": "https://ys-api.mihoyo.com/event/download_porter/link/ys_cn/official/pc_backup320",
  "downloadFileName": "yuanshen.exe"
}
```

### Описание параметров

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `enableRealDownload` | Boolean | `false` | `false` = фальшивый режим (по умолчанию), `true` = реальный режим загрузки |
| `downloadUrl` | String | miHoYo API | Целевой URL для реального режима загрузки, действует только при `enableRealDownload=true` |
| `downloadFileName` | String | `yuanshen.exe` | Имя сохраняемого файла, действует только при `enableRealDownload=true` |

### Переключение режимов

1. **Закройте игру**
2. Отредактируйте `.minecraft/config/create-download-genshin/mod_config.json`
3. Измените `enableRealDownload` на `true` (при необходимости измените `downloadUrl` для пользовательской ссылки)
4. Сохраните файл
5. **Перезапустите игру** (изменение режима требует перезапуска)

## Технические детали

### Основные механизмы

| Механизм | Реализация |
|----------|------------|
| **Обнаружение Create** | `FabricLoader.getInstance().isModLoaded("create")` |
| **Триггер** | Событие `ClientPlayConnectionEvents.JOIN` (при подключении игрока к миру) |
| **Планирование потоков** | `client.execute {}` для отложенного вызова GUI в основном потоке |
| **Асинхронная загрузка** | `CompletableFuture.runAsync` в фоновом потоке |
| **HTTP-запросы** | Встроенный Java `HttpURLConnection` (без дополнительных зависимостей) |
| **Передача прогресса** | `AtomicLong` / `AtomicBoolean` для потокобезопасного чтения/записи |
| **Null-безопасность** | Kotlin `?.`, `?:`, `coerceIn` null-безопасный синтаксис |
| **Обработка исключений** | Все операции ввода-вывода/сети/файлов обёрнуты в `try-catch` |

### Алгоритм прогресса фальшивого режима

```kotlin
// Обновление каждые 8~20 кадров (имитация非常 медленной сети, ~0.4-1 сек)
val updateInterval = 8 + random.nextInt(13)
if (tickCount % updateInterval != 0) return

// Расчёт дельты прогресса
val delta = if (random.nextDouble() < 0.08) {
    // 8% вероятность регресса: 0.05%~0.15%
    -(0.0005 + random.nextDouble() * 0.001)
} else {
    // 92% вероятность увеличения: 0.03%~0.12%
    0.0003 + random.nextDouble() * 0.0009
}

// Жёсткая блокировка на 99%, никогда не достигает 100%
progress = (progress + delta).coerceIn(0.0, 0.99)
```

### Поток событий

```
Запуск игры
  └→ CreateDownloadGenshin.onInitialize()
       ├→ ModConfig.init()          // Загрузка/создание конфигурации
       └→ FileUtil.initDataDir()    // Инициализация каталога данных

  └→ CreateDownloadGenshinClient.onInitializeClient()
       └→ ClientEvents.register()   // Регистрация слушателя события JOIN

Игрок входит в мир
  └→ Срабатывает ClientPlayConnectionEvents.JOIN
       └→ client.execute {
            └→ showDownloadScreen()
                 ├→ isCreateModLoaded()  // Обнаружение Create
                 │    └→ false → Молчаливый возврат, без действий
                 └→ true → Показать соответствующий GUI на основе конфигурации
                      ├→ FakeDownloadScreen (фальшивый режим)
                      └→ RealDownloadScreen (реальный режим)
                          └→ DownloadUtil.downloadAsync()  // Фоновая загрузка
```

## FAQ

### Q: Всплывающее окно не появляется?

1. **Подтвердите, что мод Create (Механическая сила) установлен** — этот мод показывает окно только при обнаружении Create
2. Подтвердите, что файл `.jar` этого мода находится в каталоге `.minecraft/mods/`
3. Подтвердите, что установлены `fabric-api` и `fabric-language-kotlin`
4. Проверьте игровой лог (`.minecraft/logs/latest.log`) на наличие следующего вывода:
   - `未检测到机械动力模组（Create），跳过弹窗` → Create не установлен
   - `已检测到机械动力模组（Create），准备弹出下载窗口` → Обнаружение успешно

### Q: Реальный режим загрузки показывает «Ошибка загрузки»?

- Проверьте сетевое подключение
- Подтвердите, что `downloadUrl` доступен (проверьте в браузере)
- Проверьте блокировку брандмауэром/прокси
- Просмотрите `.minecraft/logs/latest.log` для получения подробной информации об ошибке
- Распространённые коды ошибок: `403` (нет доступа), `404` (ссылка недействительна), `timeout` (таймаут сети)

### Q: Как изменить ссылку для загрузки?

Отредактируйте поле `downloadUrl` в файле конфигурации и перезапустите игру.

### Q: Будут ли изменены системные файлы?

**Нет.** Всплывающее окно фальшивого режима — это чисто визуальный эффект, не выполняющий реальных операций. Реальный режим загрузки только сохраняет файлы в каталог `.minecraft/create-download-genshin-data/`, не изменяя системные файлы.

### Q: Сервер вылетит?

**Нет.** Весь клиентский код (GUI, слушатели событий, обнаружение Create) находится в наборе исходного кода `client` и не загружается на сервере. Сервер загружает только классы управления конфигурацией и файловые утилиты.

### Q: Игра приостанавливается при открытом окне?

**Нет.** Оба всплывающих окна переопределяют `isPauseScreen()` для возврата `false`, игровой мир продолжает работать.

### Q: Можно ли использовать без установки Create?

Нет. Этот мод предназначен специально для пользователей Create. Без установленного Create мод полностью молчалив.

## Разработка

### Настройка окружения

1. Установить JDK 17+
2. Fork и клонировать этот репозиторий
3. Открыть проект в IntelliJ IDEA (рекомендуется, автоматически определяет Gradle проект)

### Стандарты кода

- Строгий null-безопасный синтаксис Kotlin, запрет на滥用 `!!`
- Все публичные API должны иметь комментарии KDoc
- Операции ввода-вывода/сети должны использовать `try-catch`, информация об исключениях записывается в лог
- Клиентский код не должен出现在 общем пакете (предотвращает вылеты сервера)
- GUI окна должны переопределять `isPauseScreen()` для возврата `false`
